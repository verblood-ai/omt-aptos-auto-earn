# Отчёт о состоянии проекта aptos_auto_earn

**Обновлено:** 2026-04-20

Документ отражает **текущее** состояние репозитория. Устаревшие пошаговые чеклисты Stage 4 перенесены в историю git (файл `TODO.md` теперь указывает на Stage 5).

---

## 1. Цель продукта

Автоматизировать участие в тестовой сети Aptos: кран (в т.ч. сценарий «с нуля»), плановые ончейн-действия (DEX), метрики, мониторинг публичных сигналов по аирдропам. Токены тестнета не имеют рыночной ценности; ценность — в воспроизводимом процессе и истории в DuckDB.

---

## 3. Модули

| Направление | Роль |
|-------------|------|
| Конфигурация | `APTOS_NETWORK`, YAML, env; при смене сети без `APTOS_NODE_URL` подставляется дефолтный fullnode |
| Кошелёк | Ключ, баланс, симуляция/отправка транзакций |
| Кран | Aptos Labs mint; лимиты и кулдаун **изолированы по сети** |
| DEX | `scripts_v2::swap`, пара APT / Pontem test USDT, классификация ошибок, ретраи только по сети |
| Метрики | `balance_history`, `transactions`, `faucet_claims`, `activity_runs`, `dex_swaps`, … |
| Аирдропы | Парсинг Currents; API-площадки — при наличии ключей |
| Оркестратор | `run.py` / планировщик, `--once` |

---

## 4. Риски

- Сброс **devnet** и пустой список модулей Liquidswap до повторной публикации пакетов.
- **Testnet faucet:** без `APTOS_FAUCET_JWT` автоматический клейм Aptos Labs недоступен (см. README / OPS).
- Парсинг внешних страниц и лимиты HTTP.

---

## 5. Качество

Unit-тесты: конфиг, DEX (сборка entry + моки simulate/submit), кран (per-network state), парсер Currents, health DuckDB; опциональный e2e при `APTOS_E2E=1`. Команда: `python -m unittest discover -s tests -v`.

---

## 6. Снимок DEX (testnet, 2026-04-20)

По REST на адресе пакета `0x190d…e12` в ABI присутствуют **`scripts_v2`** и **`router_v2`**. Тестовые монеты `0x4341…8b9::coins::USDT` используются в паре с APT. Подробности и ссылки: **[docs/DEX.md](docs/DEX.md)**.
